Git

Sistema de control de versiones:Un programa que controla o administra las distintas versiones de un programa

  • Sistema de control de versiones distribuido
  • coordinado entre multiples desarrolladores
  • Quien hizo los cambios y cuando
  • Volver atras en cualquier momento
  • Repositorio local y remoto
  • Todos los desarrolladores tienen una copia local

Conceptos

  • Hacer un seguimiento de la historia del codigo
  • Tomar "snaptshots"(instantaneos) de tus archivos
  • Tu decides cuando hacer los snapshots para guardar los cambios
  • Puedes visitar los snapshots en cualquier momento
  • Puedes hacer cambios antes de guardarlos

Comandos basicos

  • git init:vamos a crear un proyecto
  • git add 'file' :Pasar los archivos del working directory al staging area
  • git status:En que estado estan los archivos
  • git commit:Pasar de staging area al repositorio
  • git push:Subirlo a un repositorio remoto para que otros trbajen
  • git pull:Puedes traer los cambios que han hecho otros desarrolladores
  • git clone:Hacer una copia desde el sector central a nuestra pc

Hay 3 estados:

  1. Working directory:Donde se trabaja con todos los archivos
  2. Staging area:Donde se agregar los archivos para el guardado
  3. repository:
